1. 웹 검색 엔진 기본 구조
1) 웹 크로울러 (bot)
  - 이재승 연구원 개발
  - 웹 사이트의 링크를 따라 들어가며 각 사이트의 검색 가능 문장을 수집 (1~2주차)
  - depth를 지정하여 처음 주어진 url에서 부터 몇 link를 따라 들어갈지 결정 (1~2주차)
  - HTML 태그를 모두 제거, 광고로 추정되는 태그의 경우 내부 내용도 제거 (HTML 파싱 필요) (3~4주차)
  - 사이트와 문장을 tuple로 만들어 list를 반환
2) 텍스트 인덱싱
  - 김기태 수석 연구원 개발
  - 위 모듈로 부터 받은 정보를 바탕으로 one-hot encoding (1~2주차)
  - 사이트 당 단어 벡터 생성 (1~2주차)
  - indexing 정보를 파일로 save/load (load는 프로그램 실행시 한번만) (1~2주차)
3) 검색
  - 김기태 수석 연구원 개발
  - 검색어를 one-hot encoding 벡터로 변환 (3~4주차)
  - indexing 정보와 검색어 벡터의 벡터 거리 연산(norm, 혹은 기타 연산 사용 가능) (3~4주차)
  - 검색 결과 반환 (3~4주차)
4) GUI
  - 정진우 사원 개발
  - windows GUI
  - website url 창(웹 크로울러를 위함), 웹 크로울링 시작 버튼 (웹 크로울링 및 인덱싱 동작) (1~2주차)
  - 로딩 화면 (1~2주차)
  - 검색어 창, 검색 버튼 (검색 동작) (3~4주차)
  - 검색 결과 리스트 창 (3~4주차)
